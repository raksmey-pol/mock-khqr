# mock-store-api

NestJS mock merchant backend for testing KHQR checkout integration.

## Run

```bash
cp .env.example .env
npm install
npm run start:dev
```

The running service reads `mock-store-api/.env`.

Required env vars:

- `MERCHANT_ID`
- `MERCHANT_API_SIGNING_SECRET` (for create-intent signing)
- `MERCHANT_WEBHOOK_SIGNING_SECRET` (for webhook signature verification)

## Where These Secrets Come From

- `MERCHANT_API_SIGNING_SECRET`:
  - Merchant can generate once approved via `POST /api/merchant/api-signing-secret/generate`.
  - Merchant can rotate via `POST /api/merchant/api-signing-secret/rotate`.
  - Rotation keeps the previous key valid for a short grace period.

- `MERCHANT_WEBHOOK_SIGNING_SECRET`:
  - Also generated by Open Banking for the merchant.
  - Can be rotated by merchant via `POST /api/merchant/webhook/secret/rotate`.

## Responsibilities

- Signs and forwards create-intent requests to Open Banking.
- Proxies checkout status polling.
- Verifies webhook signature from Open Banking.
- Stores received webhook events in memory for quick visibility.
